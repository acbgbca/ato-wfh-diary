Plan a new feature for the ATO WFH Diary

Follow this workflow:
1. Ask about the feature that we are going to build.
2. Interpret the feature request, and ask clarifying question for anything that isn't clear or is ambiguous
3. Explorer the code so that you understand the structure of the application and have an idea of what will need to change. Ask further questions to clarify how the feature will be implemented
4. Come up with a complete implementation plan, and present that to the user for approval
5. If the user has feedback, iterate the design based on the feedback until there is approval
6. Once approved, create a GitHub issue with the details of the change. Include the justification for the change, and the complete solution

Once the GitHub issue has been created we are finished. Do not offer to start implementation, that will be done in a separate session.
